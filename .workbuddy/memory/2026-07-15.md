
- **修复 meme-weibo.vue 多根元素报错**：用户反馈 `meme-weibo.vue` 模板编译报错 "Component template should contain exactly one root element"。根因：批量修改时该文件的 `</div>` 闭合位置错误，导致 `<SiteFooter>` 落在了原根 div 之外，形成两个根元素。修复：① 在 `<template>` 后插入 `<div class="content">` 包裹层；② 将 `<SiteFooter>` 移入 `.content` 内；③ 修正缩进（`.meme-weibo` 在 `.content` 内部）；④ 在 `<style>` 中新增 `.content { padding: 20px 24px; min-width: 0; }` 包装样式（与其它已改页面保持一致）。`npm run build` 通过。

- **批量改页面后的质量检查清单**（重要经验）：① 模板必须且只能有一个根元素 —— 批量插入 SiteFooter 时若原 div 闭合位置错乱，会出现 SiteFooter 跑到根 div 外的多根元素错误；② 插入 SiteFooter 前要确保已正确闭合所有外层 div；③ 用 `<div class="content">` 包裹能同时满足"单根元素"和"统一内边距"两个需求。已扫描全项目，未发现其他同类问题。

- **重要教训**：① `git checkout -- <file>` 会丢弃工作区修改，要谨慎；② 如果只部分还原要分别 checkout/unstash；③ batch 改文件后应 `git add .` 或 `git commit` 一次，再开始新工作，避免意外丢失；④ 推荐把这次的完整布局改动做一次 commit，commit message 类似 "feat(layout): 全站接入侧栏布局"。
