- 重要坑：本项目是 `nuxt generate` 静态站模式，`dist/` 才是线上产物（tools.huhaha.vip 即此）。本地预览绝不能 `nuxt start`（SSR 模式）——其 `.nuxt/dist/client` 下无 client JS（0 个），导致所有 `/_nuxt/*.js` 404 → 页面 hydration 不执行 → `layouts/default.vue` 里 `index_page.hide{opacity:0}` 依赖 `$store.state.loaded`（仅在 mounted 设 true）永远不解除 → 整页空白。正确预览/部署流程：先 `npx nuxt generate` 重新生成 dist（注意末尾 @nuxtjs/sitemap 会报 `Invalid URL` FATAL，是项目原有 base URL 配置问题，不影响 HTML/JS 产物），再用静态服务器 serve `dist/`（如 `python -m http.server 8080`）。本次已重新 generate 并验证 dist/index.html 含新布局、dist/_nuxt 有 113 个 JS，8080 端口预览正常。
